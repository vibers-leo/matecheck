## Objective (목표)

**"모든 보금자리 구성원들이 스트레스 없이, 즐겁게 함께 살아가는 일상을 경험하게 한다"**

*→ 단순한 앱 출시를 넘어, 사용자의 삶에 실질적인 가치를 제공하는 필수 생활 플랫폼으로 자리매김*

---

## Key Results (핵심 결과)

### KR 1: **사용자 신뢰와 만족도 달성**
- **지표**: 앱스토어 평점 **4.5점 이상** (최소 100건 리뷰 기준)
- **측정 방법**:
  - App Store / Google Play 평점 모니터링
  - 사용자 피드백 키워드 분석 (긍정 키워드 70% 이상)
- **도전성**: 첫 출시 앱이 100건 리뷰를 받고 4.5점 이상을 유지하는 것은 매우 높은 목표 (70% 달성 시에도 의미 있음)

### KR 2: **실제 사용 가치 증명 (Retention)**
- **지표**: 출시 후 30일 내 **재방문율(DAU/MAU) 40% 이상** 달성
- **측정 방법**:
  - Firebase Analytics / Mixpanel로 DAU/MAU 추적
  - 주요 기능 사용 빈도 분석 (할 일 체크, 일정 등록 등)
- **도전성**: 신규 앱이 첫 달에 40% 재방문율을 달성하는 것은 프로덕트 마켓 핏(PMF)을 의미

### KR 3: **안정적이고 빠른 서비스 제공**
- **지표**:
  - 크래시 발생률 **0.5% 이하** (Crashlytics 기준)
  - API 응답 속도 **평균 200ms 이하**
  - 앱 초기 로딩 시간 **2초 이내**
- **측정 방법**:
  - Firebase Crashlytics 크래시 리포트
  - Backend API 모니터링 (New Relic / Datadog)
  - 앱 성능 프로파일링 (React Native Performance Monitor)
- **도전성**: 신규 앱에서 크래시율 0.5% 이하는 높은 품질 기준

### KR 4: **프로덕트 마켓 핏 검증**
- **지표**: 출시 후 **2주 내 최소 500명 이상 가입**, **100개 이상 활성 보금자리** 생성
- **측정 방법**:
  - 회원가입 수 (Backend Analytics)
  - 활성 Nest 수 (최소 2명 이상 구성원, 주 1회 이상 활동)
- **도전성**: 초기 마케팅 없이 자연 유입으로 500명 확보는 입소문 효과 필요

## Initiatives (실행 방안)

### Phase 1: **배포 준비 완료** (2주)
- [ ] **환경 변수 관리 체계화**
  - `.env` 파일 생성 및 `expo-constants`로 API URL, 키 관리
  - 개발/스테이징/프로덕션 환경 분리

- [ ] **에러 핸들링 강화**
  - API 오류 시 사용자 친화적 Toast 메시지 (`react-native-toast-message`)
  - 네트워크 오류 재시도 로직 구현
  - 에러 바운더리 추가 (React Error Boundary)

- [ ] **API 클라이언트 추상화**
  - `services/api.ts` 파일로 중복 코드 제거
  - Axios 또는 Fetch 래퍼 생성 (자동 인증 헤더, 리프레시 토큰 로직)

- [ ] **인증 보안 강화**
  - JWT 토큰 기반 인증 구현
  - Secure Storage 활용 (expo-secure-store)
  - 자동 로그아웃 및 세션 관리

- [ ] **성능 최적화**
  - 이미지 최적화 (`react-native-fast-image`)
  - 긴 리스트 가상화 (`@shopify/flash-list`)
  - 불필요한 리렌더링 제거 (React.memo, useMemo, useCallback)

- [ ] **앱 아이콘 및 스플래시 고화질 교체**
  - 1024x1024px 앱 아이콘 디자인
  - 브랜드 아이덴티티 반영

### Phase 2: **품질 보증 (QA)** (1주)
- [ ] **테스트 코드 작성**
  - 핵심 비즈니스 로직 유닛 테스트 (Jest)
  - 주요 화면 통합 테스트 (React Native Testing Library)
  - E2E 테스트 (Detox - 선택 사항)

- [ ] **크래시 모니터링 연동**
  - Firebase Crashlytics 설정
  - Sentry 연동 (에러 추적 및 알림)

- [ ] **베타 테스트**
  - TestFlight (iOS) / 내부 테스트 (Android) 배포
  - 최소 10명 이상 베타 테스터 피드백 수집
  - 주요 버그 수정 및 UX 개선

### Phase 3: **스토어 배포 및 런칭** (1주)
- [ ] **개인정보 처리방침 작성**
  - 노션/블로그에 작성 및 앱 내 링크 추가
  - 필수 동의 항목 명시

- [ ] **앱 스토어 최적화 (ASO)**
  - 매력적인 앱 이름 및 부제목
  - 핵심 키워드 선정 ("가족 일정 공유", "룸메이트 관리", "공동 가계부")
  - 스크린샷 4-8장 준비 (주요 기능 강조)
  - 앱 설명 작성 (가치 중심, 짧고 명확하게)

- [ ] **EAS Build로 프로덕션 빌드**
  - iOS: AAB 빌드 및 App Store Connect 제출
  - Android: AAB 빌드 및 Google Play Console 제출

- [ ] **심사 제출 및 모니터링**
  - Apple/Google 심사 대응
  - 반려 시 빠른 수정 및 재제출

### Phase 4: **런칭 후 모니터링 및 개선** (지속)
- [ ] **사용자 분석 도구 연동**
  - Firebase Analytics / Mixpanel 이벤트 추적
  - 핵심 지표 대시보드 구축 (DAU, MAU, Retention, Churn)

- [ ] **사용자 피드백 수집**
  - 앱 내 피드백 기능 (`support_tickets`)
  - 앱스토어 리뷰 모니터링 및 답변
  - 소셜 미디어/커뮤니티 반응 추적

- [ ] **데이터 기반 개선**
  - A/B 테스트 (온보딩 플로우, 주요 CTA 등)
  - 사용자 행동 분석 (어떤 기능을 가장 많이 사용하는가?)
  - 이탈 구간 파악 및 개선

- [ ] **빠른 이터레이션**
  - 주 1회 마이너 업데이트 (버그 수정, UX 개선)
  - 2주 1회 기능 업데이트 (사용자 요청 기능 추가)

## OKR 자체 평가 (Self-Correction)

### ❌ 기존 방식 (할 일 목록형 - Bad)
- "백엔드 API 개발 완료"
- "프론트엔드 UI 구현"
- "테스트 코드 작성"

→ **문제점**: 무엇을 했는가(Output)만 나열되어 있고, 사용자에게 어떤 가치를 주었는가(Outcome)가 없음.

### ✅ 현재 OKR (비즈니스 가치 중심 - Good)
- **Objective**: "모든 보금자리 구성원들이 스트레스 없이, 즐겁게 함께 살아가는 일상을 경험하게 한다"
  - → 사용자 중심, 감성적, 팀의 가슴을 뛰게 함

- **Key Results**: 앱스토어 평점 4.5점, 재방문율 40%, 크래시율 0.5% 이하 등
  - → 구체적이고 측정 가능하며, 70% 달성만 해도 성공이라 느낄 만큼 도전적

- **Initiatives**: 실제로 KR을 달성하기 위해 수행할 구체적인 작업
  - → Output이지만, Outcome을 위한 수단으로 명확히 정의됨

## 성공 기준

### 최소 성공 (70% 달성)
- 앱스토어 평점: **4.2점** (KR1의 93%)
- 재방문율: **28%** (KR2의 70%)
- 크래시율: **0.7%** (KR3의 80%)
- 가입자 수: **350명** (KR4의 70%)

### 완전 성공 (100% 달성)
- 모든 KR 지표 100% 달성
- 사용자들의 자발적인 SNS 공유 및 입소문 확산
- 언론/블로그에서 긍정적인 리뷰 게재

### 초과 달성 (120% 이상)
- 앱스토어 평점: **4.7점 이상**
- 재방문율: **50% 이상**
- 가입자 수: **1000명 이상**
- **투자 유치 또는 파트너십 제안** 받음

## OKR 점검 주기

- **주간 체크인**: 매주 월요일, Initiatives 진행 상황 점검
- **2주 리뷰**: 2주마다 KR 지표 추이 분석
- **월간 회고**: 매월 마지막 주, OKR 달성률 평가 및 다음 달 계획 수립

---

## OKR 운영 원칙

1. **투명성**: 모든 팀원이 OKR을 보고 이해할 수 있어야 함
2. **자율성**: Top-down이 아닌, 팀원들이 스스로 Initiatives를 제안하고 실행
3. **도전성**: 100% 달성이 쉬운 목표는 좋은 OKR이 아님 (70% 달성 = 성공)
4. **측정 가능**: "열심히 했다"가 아니라 "숫자로 증명"
5. **정렬(Alignment)**: 모든 작업이 Objective를 향해야 함
